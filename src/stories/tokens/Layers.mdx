# Layers

Z-index controls stacking order. Tailwind v4 provides utilities `z-0` through `z-50` as fixed steps. No custom property tokens are defined — use the utility classes directly or extend with `@theme`.

## Conventions

- Keep overlay backdrops at `z-40` and the content they cover at `z-50` so they always layer correctly.
- Toasts and notifications sit at `z-50` — they must appear above everything else.
- Never use arbitrary high z-indices (e.g. `z-[9999]`) — extend the scale in your Tailwind config instead.
- Popover/tooltip anchoring libraries typically manage their own z-index via a portal; confirm they sit in the `z-20`–`z-30` range.

## Extending the Scale

Add extra steps in `app.css` with `@theme`:

```css
@theme {
  --z-60: 60;
  --z-70: 70;
}
```

Then use `z-60`, `z-70` as Tailwind utilities.
